## 详细配置说明

### 1. 计算单元

<Callout type="tip">
  计算单元是系统的核心处理单元，负责运行各种AI模型和推理框架
</Callout>

#### 推理框架配置
- **vLLM推理框架**
  - 大语言模型（LLM）服务
  - 端口：58000
- **TEI Embedding模型**
  - 端口：58080
- **TEI Reranker模型**
  - 端口：58081

#### 存储路径配置
<Tabs items={['手动配置模型存储', '自动拉起模型存储']}>
  <Tab>
    ```
    ~/cfe/model/
    ├── llm/          # 大语言模型
    ├── embedding/    # Embedding模型
    └── reranker/     # Reranker模型
    ```
  </Tab>
  <Tab>
    ```
    ~/cfe/autoModel/
    ├── llm/          # 大语言模型
    ├── embedding/    # Embedding模型
    └── reranker/     # Reranker模型
    ```
  </Tab>
</Tabs>

#### 自动拉起配置
- 路径：`~/cfe/autorun.sh`
- 说明：非加密自动拉起脚本，用户可编辑，用于指定默认自动启动的模型脚本

### 2. 应用单元

<Callout type="tip">
  应用单元负责运行和管理各类应用服务，提供灵活的应用部署方案
</Callout>

#### 服务配置
- **Nginx**
  - 用户应用服务
  - 端口：80
- **系统平台**
  - Dify：端口59080
  - Altai（本地部署版本）：端口59299

#### 存储路径配置
<Tabs items={['导入应用存储', '运行应用存储']}>
  <Tab>
    ```
    ~/cfe/app/    # 导入非加密应用的存储位置
    ```
  </Tab>
  <Tab>
    ```
    ~/app/        # 运行应用的存储位置（位于应用单元的NVMe硬盘）
    ```
  </Tab>
</Tabs>

### 3. 硬件控制单元

<Callout type="tip">
  硬件控制单元提供全面的硬件监控和配置功能，确保系统稳定运行
</Callout>

#### 存储路径配置
- **硬件监控前端**
  - 路径：`/sdcard/web`
  - 说明：用户可自行修改硬件监控的前端界面
- **矩阵显示Logo配置**
  - 路径：`/sdcard/matrix.json`
  - 说明：用于配置矩阵显示的Logo
